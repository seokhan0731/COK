## 협업 규칙 (Convention)

### Branch Convention
- main: 배포용 브랜치(직접 Push 금지)
- develop: 개발 통합용 브랜치
- feature/[기능명]: 개인 작업 브랜치 (Ex. feature/login)

### Commit Convention
- test: 테스트 코드 추가 및 수정
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- refactor: 코드 리팩토링
- chore: 4가지 태그 외 빌드 수정 업무 등
