## 1. 역량 측정

역량 측정은 사용자의 프로필 생성시, 입력 받은 정보와 설문을 토대로 측정한다.

| 역량    | 설문 가중치 | 자격증 가중치 | 백준 가중치 |
|:------|:-------|:--------|:-------|
| 구현력   | 98     | 2       | 0      |
| 협업    | 100    | 0       | 0      |
| 트렌드   | 95     | 5       | 0      |
| CS 지식 | 90     | 10      | 0      |
| 알고리즘  | 25     | 5       | 70     |
| 인프라   | 95     | 5       | 0      |

### 역량별 가중치 설정 관련

**1. 구현력**  
구현력은 실제 설계·디버깅·예외처리 능력이 중요하며 자격증만으로 검증하기 어렵기 때문에
설문 중심으로 평가하였다.

**2. 협업**  
협업 역량은 코드 리뷰, 의사소통, 갈등 해결 등 행동 기반 특성이므로 자격증을 통한 객관적 검증이 어렵다고 판단하여 설문만 반영하였다.

**3. 트렌드**  
트렌드 역량은 새로운 기술에 대한 학습 태도가 핵심이므로 설문 중심으로 측정하였으며,
최신 기술 관련 자격증 취득 여부를 보조적으로 반영하였다.

**4. CS 지식**  
CS 역량은 국가공인 자격증을 통해 운영체제·네트워크·데이터베이스 기초지식을
객관적으로 검증할 수 있으므로 자격증 비중을 가장 높게 설정하였다.

**5. 알고리즘**  
알고리즘 역량은 실제 문제 해결 능력이 핵심이므로 백준 티어를 주 지표로 활용하였으며,
자격증은 이론 학습 여부를 보조적으로 반영하였다.  
**[백준을 하지 않는 경우는 내부 평가 요소에서 재정규화 예정]**

**6. 인프라**  
인프라 자격증은 클라우드 및 시스템 운영 지식을 검증할 수 있으나
실제 배포 및 운영 경험을 직접 증명하지는 못하므로 보조 지표로 활용하였다.

## 2. 직무 추천

직무 추천은 측정된 역량과 경험 지표를 토대로 진행된다.  
경험 지표는 사용자의 대표 Git 리포지토리 중, 사용된 기술스택을 통해 측정한다.

**(역량 지표*0.6)+(경험 지표*0.4)**  
경험 점수와 역량 점수의 비율은 40:60으로 설정하였다.  
이는 특정 기술 스택 사용 경험만으로는 개발자의 실제 문제 해결 능력과 기본 역량을 충분히 판단하기 어렵다고 보았기 때문이다.  
다만 기술 스택 사용 경험 또한 직무 적합도를 판단하는 중요한 요소이므로,
특정 직무 관련 기술 스택 경험이 전혀 없는 사용자가 추천되는 문제를 방지하기 위해 경험 점수 역시 함께 반영하였다.

**1. 백엔드 개발자**  
백엔드 개발자의 핵심 업무는 비즈니스 로직 구현, 데이터 처리, API 설계 및 서버 개발이다.  
따라서 실제 기능을 구현하는 능력인 구현력을 가장 중요한 역량으로 설정하였다.  
또한 데이터베이스, 네트워크, 운영체제, 트랜잭션과 같은 시스템 전반에 대한 이해가 필수적이므로 CS 지식에 높은 비중을 부여하였다.  
알고리즘은 대용량 데이터 처리 및 효율적인 로직 설계에 필요하므로 중간 수준의 비중을 부여하였다.  
협업과 인프라는 실무에서 중요한 요소지만 직무의 핵심 업무는 아니므로 보조적인 비중으로 설정하였다.  
트렌드 역량 역시 중요하지만 최신 기술 습득보다 서비스 구현 능력이 우선이라고 판단하여 가장 낮은 비중을 적용하였다.

**2. 프론트엔드 개발자**  
프론트엔드 개발자는 사용자가 직접 사용하는 화면을 구현하는 직무이다.  
따라서 UI 구현, 상태 관리, 컴포넌트 설계 등의 능력을 의미하는 구현력을 가장 높은 비중으로 설정하였다.  
또한 디자이너, 기획자, 백엔드 개발자와 긴밀하게 협업해야 하므로 협업 역량을 중요하게 반영하였다.  
React, Vue, Next.js 등 프론트엔드 기술 생태계는 변화 속도가 빠르기 때문에 새로운 기술을 학습하고 적용하는 트렌드 역량 또한 높은 비중을 부여하였다.  
CS 지식과 알고리즘은 필요하지만 핵심 업무는 아니므로 보조 수준으로 반영하였다.

**3. 모바일 개발자**  
모바일 개발자는 실제 애플리케이션을 구현하는 능력이 가장 중요하므로 구현력을 최우선으로 설정하였다.  
또한 Flutter, Swift, Kotlin 등 모바일 기술 환경은 빠르게 변화하므로 트렌드 역량에 높은 비중을 부여하였다.  
모바일 환경에서는 메모리 관리, 앱 생명주기, 비동기 처리 등에 대한 이해가 필요하기 때문에 CS 지식 역시 중요하게 반영하였다.

**4. AI/ML 엔지니어**  
AI 직무는 모델 설계, 학습, 최적화 등 문제 해결 중심 업무가 많기 때문에 알고리즘 역량을 가장 높게 설정하였다.  
또한 실제 서비스에서는 모델 개발뿐 아니라 데이터 처리, 모델 서빙, API 연동 등이 필요하므로 구현력도 높은 비중을 부여하였다.  
AI 분야는 기술 발전 속도가 매우 빠르기 때문에 최신 논문과 라이브러리를 학습하는 트렌드 역량 역시 중요하게 반영하였다

**5. DevOps 엔지니어**  
DevOps 직무의 핵심은 서비스 운영과 배포 자동화이다.  
Docker, Kubernetes, CI/CD, 클라우드 환경을 다루는 능력이 직무의 중심이므로 인프라 역량에 가장 높은 비중을 부여하였다.  
또한 운영체제와 네트워크에 대한 이해가 필수적이므로 CS 지식을 두 번째 핵심 역량으로 설정하였다.  
DevOps는 개발팀 전체를 지원하는 역할을 수행하므로 협업 역량 역시 중요하게 반영하였다.

**6. 데이터 엔지니어**  
데이터 엔지니어는 데이터 파이프라인 구축과 대규모 데이터 처리 환경을 설계하는 직무이다.  
따라서 데이터베이스, 분산처리, 저장구조에 대한 이해를 포함하는 CS 지식을 가장 중요하게 설정하였다.  
또한 Hadoop, Spark, Airflow 등 데이터 인프라를 구축하는 능력이 중요하므로 인프라 역량에 높은 비중을 부여하였다.

**7. PM/매니지먼트**  
PM은 직접 개발을 수행하는 직무가 아니라 프로젝트를 조율하고 방향을 결정하는 역할이다.  
따라서 개발자, 디자이너, 이해관계자 간의 의사소통 능력이 가장 중요하므로 협업 역량을 최우선으로 설정하였다.  
또한 시장 변화와 사용자 요구를 분석해야 하므로 트렌드 역량에도 높은 비중을 부여하였다.  
기술적 의사결정을 이해하기 위한 기본적인 개발 지식은 필요하지만 실제 구현 능력이나 알고리즘 역량은 상대적으로 낮은 비중으로 설정하였다.

**8. 시스템/임베디드 개발자**  
시스템 및 임베디드 개발은 하드웨어와 소프트웨어가 밀접하게 연결된 분야이다.  
운영체제, 메모리 구조, 프로세스 관리, 하드웨어 제어에 대한 이해가 필수적이므로 CS 지식을 가장 중요하게 설정하였다.  
또한 저수준 코드 구현 능력이 중요하므로 구현력에도 높은 비중을 부여하였다.

**9. 그래픽스 엔지니어**  
그래픽스 엔지니어는 렌더링, 쉐이더, 물리 연산 등 수학적 계산과 최적화가 핵심인 직무이다.  
따라서 문제 해결 능력을 의미하는 알고리즘 역량을 가장 높은 비중으로 설정하였다.  
또한 OpenGL, Vulkan, Unity 등을 활용한 실제 구현 능력이 중요하므로 구현력에도 높은 비중을 부여하였다.

**10. 블록체인 엔지니어**  
블록체인 개발은 분산 시스템, 네트워크, 보안, 암호학에 대한 이해가 중요하다.  
따라서 CS 지식을 가장 높은 비중으로 설정하였다.  
또한 합의 알고리즘과 스마트 컨트랙트 설계에는 문제 해결 능력이 요구되므로 알고리즘 역량을 두 번째 핵심 역량으로 설정하였다.  
블록체인 생태계는 변화 속도가 매우 빠르므로 트렌드 역량 역시 중요하게 반영하였다.

**11. 영상/음성 엔지니어**  
영상 및 음성 처리 분야는 신호 처리, 압축, 객체 인식, 음성 분석 등 계산 중심의 업무 비중이 높다.  
따라서 알고리즘 역량을 가장 중요한 요소로 설정하였다.  
또한 OpenCV, FFmpeg 등의 라이브러리를 활용하여 실제 서비스를 구현하는 능력이 중요하므로 구현력에 높은 비중을 부여하였다.
