## 기술 스택 선정 기준

본 서비스는 Github Repository 기반으로 사용자의 실제 개발 경험을 분석하여 직무 경험 점수를 산출하는 것을 목표로 한다.  
따라서 기술 스택 선정 시 다음 기준을 적용하였다.

**1. 높은 사용 빈도를 가지는 기술 스택**  
대학생 팀 프로젝트 및 현업에서 자주 사용되는 기술 스택을 우선 선정하였다.  
예를 들어 Spring Boot, React, Docker, TensorFlow 등은 각 직무 분야에서 높은 활용도를 가지므로 핵심 기술 스택으로 포함하였다.

**2. 특정 직무를 대표할 수 있는 기술 스택**
각 기술 스택이 특정 직무의 핵심 업무와 직접적으로 연결되는지를 기준으로 선정하였다.  
예를 들어 Spring Boot는 백엔드 직무, React는 프론트엔드 직무, Kubernetes는 인프라 직무의 대표 기술 스택으로 판단하였다.

**3. 직무 간 교집합이 실제로 발생하는 기술 스택 반영**  
일부 기술 스택은 하나의 직무에만 사용되지 않기 때문에, 실제 산업 환경에서의 활용 범위를 고려하여 서포트 직무를 함께 부여하였다.  
예를 들어 Docker는 인프라 직무의 핵심 기술이지만 백엔드 및 데이터 환경 구성에도 자주 사용되므로 서포트 직무로 함께 반영하였다.
