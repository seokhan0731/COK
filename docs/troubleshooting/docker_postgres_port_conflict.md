# [Docker] 로컬 PostgreSQL 포트 충돌(5432) 이슈

## 1. 개요
* **날짜:** 2026.05.13
* **작성자:** 김석환
* **관련 시스템/기능:** 도커 DB compose 파일
* **핵심 요약:** 컨테이너는 정상적으로 올라간, pgAdmin을 통해 접속시 접속이 안 되는 문제 발생

## 2. 현상 및 상황
* **문제 발생 환경:** 로컬
* **현상:** 
  * docker compose up -d -> FATAL: password authentication failed for user "유저명"

## 3. 시도했던 해결 방안
* **가설1:** pgAdmin을 통해 접속시, 입력값 오타로 인한 접속 실패 -> **결과:** [실패] 여러 차례 확인 후, 재입력해도 실패
* **가설2:** .env 속 내용 미반영 -> **결과:** [실패] docker compose config를 통해 확인, 해당 가설이 아닌 것을 확인
* **가설3:** 로컬 5432 포트 선점 -> **결과:** [성공] 호스트 포트 번호를 54321로 수정 후, 재접속 시도

## 4. 원인 분석
* 로컬 윈도우 환경에 설치되어있던 PostgreSQL이 이미 5432 포트를 선정하고 있었음.
* 외부에서 5432 포트로 접속을 시도할 때, 도커가 아닌 로컬 DB가 이를 가로채어 발생한 오류.

## 5. 해결 방안
* 'docker-compose.yml'에서 호스트 매핑 포트를 54321로 변경
  * 변경 전:     'ports: - "5432:5432"'
  * 변경 후:     'ports: - "54321:5432"'
