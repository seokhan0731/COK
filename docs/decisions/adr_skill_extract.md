# ADR 003: 기술 스택 자동 탐지 범위 제한

## 개요

* **날짜:** 2026.06.11
* **작성자:** 김석환

## 컨텍스트

* **배경:**  
  Github API를 이용한 패키지 관리 파일 기반 분석만으로는 Unity 등 프로젝트 전반의 소스 코드를 분석해야 식별 가능한 기술 스택 탐지 불가
* **요구 사항:**
* 해당 기술 스택을 제거하면, 시스템, 그래픽스 엔지니어의 경험 지수를 추출할 항목이 거의 없어지기에, 반영하는 방법 필요

## 결정

* 자동 탐색된 기술 스택과, 탐색할 수 없는 기술 스택 분리
    1. package.json, build.gradle, requirements.txt 등의 패키지 관리 파일과, 파일 유무만 확인하여 탐색 가능한 Docker 등의 기술 스택은 자동 탐색
    2. Unity 등 특정 파일이 아닌 모든 파일에서 적용 가능한 기술 스택은, 사용자에게 별도로 체크하도록 유도

## 기대 효과

* **Github 호출 비용 감소:** 어떻게든 탐색을 하고자 한다면, 모든 파일을 api를 통해 조회해야 했지만, 별도 체크 유도를 통해 호출 비용을 감소시킬 수 있음
* **포괄적 직무 유지 가능:** 해당 기술 스택이 경험 지수로 들어가는 직무들을 포기하지 않음으로써, 직무 추천에 대한 포괄성을 유지할 수 있음
