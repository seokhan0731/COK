# ADR 002: 룩업 테이블 일부 제거

## 개요

* **날짜:** 2026.05.29
* **작성자:** 김석환

## 컨텍스트

* **배경:**  
  여러 가지 항목의 점수가 부여된 현재 기획에서, 룩업 테이블 참조를 위해 발생하는 I/O 비용 우려
* **요구 사항:**
* DB에서 반드시 관리 안 해도 되는 값과 룩업 테이블 제거

## 결정

* 기술스택 관련 테이블 제거 및 Enum 클래스로 대체
    1. 수정된 기획에선, 사용자 리포지토리 분석에 따른 기술스택만을 보여주기에 기술 스택이 DB에서 관리할 필요가 없다고 판단
    2. 1.에 따라, MasterTechStack, TechStackJobWeight, UserTechStack 테이블 제거
* 백준 티어 관련 테이블 미생성
    1. 백준 티어에 따라 부여되는 점수는 위와 같은 배경에서 DB 저장이 불필요하다고 판단
    2. 역량값 계산시, Enum 클래스와 User 테이블에서의 algorithm_level 속성을 갖고와서 사용하는 것이 유리할 것이라 판단
* 타 룩업 테이블 구조 유지
    1. 마스터 자격증, 설문 문항, 알고리즘 레벨, 원티드 태그 등은 사용자에게 보여줘야 되는 정보이기에,  이에 대한 룩업 테이블은 한 개의 영역에서 관리하는 것이 유리할 것이라 판단

## 기대 효과

* **I/O 비용 감소:** 일부 룩업 테이블을 제거함으로써, DB 접근을 감소에 따른 비용 절감 가능
* **명확한 책임 영역:** DB에 저장해야 되는 값은 별도로 Enum을 사용 안 함에 따라, 유지보수에 유리한 구조로 구현 가능
