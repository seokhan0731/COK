# ADR 001: 직무 추천 로직 수정

## 개요

* **날짜:** 2026.05.28
* **작성자:** 김석환

## 컨텍스트

* **배경:**  
  중간발표 피드백(설문 문항 최소화 및 직무 추천 로직 수정) 수용
* **요구 사항:**
* 기존 더블 메트릭스 구조(문항에 따른 역량+직무 점수 부여)를 문항+GitHub로 수정
* 직무 추천 로직에 경험 지표 추가(GitHub 사용)

## 결정

* 역량 분석 및 직무 추천 로직 재설계
    1. 기존에는 각 문항들이 역량과 직무에 동시에 영향을 미쳤지만, 문항은 역량에만 영향을 미치고, 역량이 직무 추천에 영향을 미치도록 구조 설계
    2. GitHub API를 통해, 사용자의 대표 리포지토리에서 사용된 기술 스택을 통해 점수를 직무 추천에 반영(경험 지표)

## 기대 효과

* **서비스 컨셉 구체화:** 직무 추천을 위해, 사용자의 흥미를 측정하는 문항을 제거함으로써, 현재 역량 기반 직무 추천이라는 서비스 컨셉이 분명해짐
* **설문 문항 감소:** 설문 문항은 사용자의 역량 측정을 위한 문항만 존재하기에, 설문을 최소화하라는 피드백 수용 가능
* **직무 추천 로직 입체화:** 설문을 통한 간접 추론을 넘어 실제 사용한 기술 스택 데이터를 융합함으로써, 입체적인 추천 알고리즘 완성 가능
