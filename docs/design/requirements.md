# 기능 요구사항

## 개요

* **날짜:** 2026.05.13
* **작성자:** 김석환

---

## 기능적 요구사항

### 1. 직무 공고(크롤링)

- [ ] 시스템은 원티드 플랫폼에서 직무 공고 데이터를 크롤링한다.
- [ ] 수집된 데이터는 사전에 정의된 스키마에 맞게 정형화되어 DB에 적재되어야 한다.
- [ ] 이미 적재된 공고와 동일한 공고는 무시한다.

### 2. 마스터 직무

- [ ] 공고의 직무는 마스터 직무에 매핑시켜야 한다.

### 3. 공고 매칭

- [ ] 시스템은 사전에 정의된 객관식/주관식 설문 문항을 제공한다.

* 역량
    *
        - [ ] 시스템은 사용자의 객관식 입력에 따라, 현재 역량 수치를 제공한다.
    *
        - [ ] 시스템은 객관식 설문 문항에 따라, 카테고리별 역량을 계산한다.
* 직무 및 공고 추천
    *
        - [ ] 시스템은 사용자의 객관식 입력에 따라, 직무 추천을 제공한다.
    *
        - [ ] 시스템은 추천된 직무와 사용자의 주관식 입력에 따라, 공고 추천을 제공한다.

### 4. 로드맵

- [ ] 시스템은 사용자의 기본 정보와 최근 설문값을 기반으로 4개월의 정해진 카테고리별 로드맵을 제공한다.
- [ ] 사용자는 미션을 완료 처리할 수 있다.
- [ ] 사용자는 월별/전체 진행도를 확인할 수 있다.
- [ ] 시스템은 각 월별 AI 코멘트를 제공해야 한다.
- [ ] 사용자는 해당 월의 미션 완수 여부와는 별개로 다음달로 넘어갈 수 있어야 한다.

### 5. 로그인/프로필 생성

- [ ] 시스템은 간편 로그인을 통해, 사용자 인증을 진행한다.
- [ ] 사용자는 기본 인적 정보와 역량 세부 정보를 입력한 후, 서비스를 이용할 수 있다.

### 6. 조회

- [ ] 시스템은 마스터 직무 정보만을 별도로 제공할 수 있다.
- [ ] 시스템은 자격증 정보만을 별도로 제공할 수 있다.

### 7. 마이페이지

- [ ] 사용자는 인적 사항 및 역량 세부 정보를 수정할 수 있다.
- [ ] 사용자는 사전에 정의된 개수만큼의 공고 매칭 결과값을 확인할 수 있다.
